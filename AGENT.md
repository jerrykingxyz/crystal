# Agent 指南

本仓库倾向于使用小而直接、无额外依赖的 shell 脚本，并通过明确的项目约定保持一致性。改动应保持窄范围、可读，并贴合当前包结构。

## 语言

- 本项目的文档和代码注释统一使用中文。
- 变量名、函数名、文件名仍使用英文，并遵循对应语言或平台的常见风格。
- 提交信息使用英文。

## 文档

- 如果代码或目录结构改动影响了已记录的行为，需要在同一次改动中更新相关文档。
- 如果引入新的项目约定，应记录到最接近的长期文档中，例如框架级规则写入 `framework.md`。
- 迁移后不要留下指向旧路径、旧命令或旧 action 语义的文档。

## 代码注释

- 只添加极简且必要的注释，用来说明意图、边界或不明显的行为。
- 可复用 helper 或包 action 需要注释时，优先写函数层面的简短注释。
- 复杂逻辑可以在关键步骤前添加短注释，帮助读者理解执行顺序。
- 不要写只是复述代码本身的注释。

## Shell 写法

- 简单的一次性 shell 逻辑保持内联。
- 只有逻辑被复用、存在清晰的包 action 边界，或复杂到命名后更易读时，才抽成函数。
- 不要为了让短脚本显得更结构化而额外抽 helper。
- 优先使用平台原生命令和仓库已有 helper，不引入新的框架依赖。

## 范围

- 做聚焦迁移时，不要顺手重写无关包。
- 本地运行文件、生成配置和 secrets 不进 git。
